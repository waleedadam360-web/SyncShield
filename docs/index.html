What is Rsync?

Rsync is a powerful utility for file synchronization between systems, commonly used for backups, mirroring, and remote transfers.

Key Features of Rsync

  • Fast, incremental backups
  • Works over SSH or network
  • Preserves permissions, timestamps, and ownership
  • Supports compression and encryption

Basic Syntax

rsync -av /source/ user@host:/destination/

-a: Archive mode (preserves permissions, etc.)
-v: Verbose output

CVE-2018-5764: Remote Command Execution

This vulnerability occurs when Rsync is executed without the --protect-args flag. If user input contains shell metacharacters, it can be interpreted as commands on the destination server.

Danger: An attacker could execute arbitrary commands remotely, potentially gaining full control of the server.

Unsafe Command

rsync -av /src user@host:/dest

⚠️ Missing --protect-args → Risk of command injection

Secure Command

rsync -av --protect-args /src user@host:/dest

✅ With --protect-args → Safe transfer

Network: Transmits data securely via SSH
Destination Server: Executes received commands

Potential Risks & Threats

If not properly secured, Rsync can lead to serious security breaches.

Remote Command Execution

An attacker can run any command on the destination server, including deleting files, installing malware, or stealing data.

Data Integrity Loss

Malicious commands can corrupt or delete critical system files during sync.

Privilege Escalation

If the Rsync process runs with elevated privileges, attackers can gain root access.

Denial of Service (DoS)

Attackers can flood the server with commands, making it unresponsive.
